The "Neuronal" Processing Unit - NonPU

Spiking Neuronal Networks, Not Artificial Neural Networks.

Prerequisites Before Reading

This is not Machine Learning. This is Biological Simulation.

The current industry focuses on Artificial Neural Networks, Large Language Models, machine learning algorithms, and reinforcement learning. These are statistical abstractions that run on Graphical Processing Units. Whether using an NVIDIA H100 or a GTX 5000, whether running different models or frameworks, the underlying physical logic circuit remains identical. The GPU executes the same matrix multiplications, consumes the same power, and operates at the same architectural level. When a language model is asked what zero multiplied by zero equals, the GPU burns its full capacity calculating that answer. When asked to describe the color of a shirt, it burns the same energy. The hardware does not distinguish between trivial and complex queries. It calculates everything with equal computational expense.

My work diverges from this approach. I am not interested in matrix multiplication or copying pre-trained models and running inference on Google Colab or remote data centers using the same logic circuits. My work focuses on a different paradigm entirely.

When searching for "Neuronal Processing Unit" or "Non-PU" on Google, Bing, Yahoo, or any search engine as of 2026, no results appear. This terminology does not exist in current research literature. I am coining this term to mark a fundamental architectural and philosophical departure from the industry-standard Neural Processing Unit.

The industry standard NPU is designed for machine learning. It relies on matrix multiplications, backpropagation, and reinforcement learning algorithms running on static data. It is a mathematical abstraction. The Non-PU is designed for Biological Simulation. It does not use machine learning in the conventional sense. It does not use matrix multiplications. It does not use reinforcement learning as an algorithmic training procedure. Instead, it relies on Spiking Neural Networks and the dynamic weighting of neurons through synaptic plasticity. It uses differential equations to model the voltage and timing of actual biological neurons.

This is not a calculator. This is a synthetic brain.

An Artificial Neural Network calculates. It takes a static snapshot of numbers, multiplies them by learned weights, and produces an output. It has no concept of time. It has no concept of physical dynamics. It is purely mathematical abstraction.

A Spiking Neuronal Network simulates. My system implements the Izhikevich neuron model, solving differential equations in real time. Each neuron has a membrane voltage that rises and falls. Each neuron has a refractory period where it cannot fire again, just like biological neurons. Each neuron communicates through discrete electrical spikes, not through continuous numerical values. This is not an approximation of intelligence through statistics. This is a replication of the physics of neural computation.

When an ANN processes an image, it calculates the probability distribution of what that image represents. When the Non-PU processes an image, it simulates the physical dynamics of a brain circuit reacting to that image in real time.

Organizations like NASA and defense research agencies are interested in neuromorphic computing because of energy efficiency.

A GPU consumes approximately 300 watts of power continuously, regardless of the computational workload. If the input is a completely black image, the GPU still calculates zero multiplied by zero millions of times per second. It burns energy calculating nothing. This is architectural waste.

The Non-PU is event-driven. If a region of an image is black, no neurons in that region spike. No spikes means no computation. No computation means no energy consumption. The system only expends power when information actually changes. For applications like CubeSats operating on limited battery power, brain-computer interfaces, edge devices, or any system where energy constraints matter, this architectural difference matters.

The human brain consumes roughly 20 watts. It does this not because neurons are individually efficient, but because the brain only activates the circuits it needs, when it needs them. When you glance at a pile of clothes and identify their color, your visual cortex processes that information in milliseconds without engaging the computational machinery required for complex numerical problems. The architecture is task-adaptive. The Non-PU replicates this principle.

One of my demonstration systems involves associative memory with image repair. If you show the system a corrupted or partial image, it reconstructs the complete memory. This differs from how machine learning image inpainting works.

A machine learning system trained on millions of images will hallucinate statistically likely pixels to fill in missing regions. It is guessing based on patterns it has seen before. My system does not guess. It has encoded the memory as a stable state in an energy landscape. When you present a corrupted image, you are placing the system near the edge of a valley. The physical dynamics of the spiking network naturally settle into the stable attractor state, which is the complete memory. This is how biological memory retrieval is theorized to function. The network does not predict. It physically converges.

I am designing the hardware architecture itself. I am defining how memory flows through the system, how parallel processing occurs, how logic gates connect, and how neurons communicate through timed electrical spikes. I am building a Neuronal Processing Unit from first principles using FPGAs, simulating the differential equations that govern biological neurons, and implementing synaptic plasticity through Spike-Timing-Dependent Plasticity.

Spike-Timing-Dependent Plasticity is Hebbian learning. It is the biological principle that neurons that fire together, wire together. It happens locally at each synapse based on the precise timing of spikes, not globally through a software optimization algorithm. This is not reinforcement learning in the machine learning sense. This is how biological brains learn.
