Reasons for Using AC415 and Its Resource Constraints

The implementation runs on an AC415 development board featuring the Intel Altera Cyclone IV EP4CE15 FPGA. This chip provides 15,408 logic elements total, 504 kilobits of embedded memory, and 56 hardware multipliers. The board includes 32 megabytes of external SDRAMThis external memory stores the 256 neuron states and the 256x256 synaptic weight matrix (64KB) during network operation and VGAThis implementation outputs real-time visualization of the 16x16 network activity through VGA to monitor pattern completion output capability for visualization.

Figure 2: Photo of AC415 development board with Cyclone IV FPGA

These constraints shape the entire architectural approach. A single Izhikevich neuron implemented directly in hardware requires differential equation solvers, multiplication units, state registers for membrane voltage and recovery variables, and threshold detection logic. Conservative estimates place the resource cost at 200 to 300 logic elementsThe Cyclone IV has 15,408 total. This implementation uses 10,200 for the neuronal sheet, 980 for the modulator, and 3,400 for infrastructure per neuron for a fully parallel implementation. Building even 50 neurons in parallel would consume the entire chip, leaving no resources for synaptic logic, memory interfaces, or output controllers.

Although the Terasic DE10-Nano with Intel Altera Cyclone V FPGA is fully capable of simulating a much higher number of neurons, we have chosen the Cyclone IV over the Cyclone V to test this in an absolutely constrained environment and to demonstrate its energy efficiency. However, the Cyclone V is the minimum viable option for my bio-computing researchClick to Read the Article Published on a Non-profit Magazine because it contains 110,000 logic elements, a 500 MHz logic fabric, and, most importantly, a system-on-a-chip architecture. This allows us to run a separate operating system for space deployment, satellites, and other HLS tasks.

Time-multiplexingInstead of building 256 parallel neurons, this implementation uses a single pipeline that processes each neuron sequentially from SDRAM, enabling 256 neurons with limited logic elements solves this resource limitation by trading silicon area for computation time. Instead of instantiating thousands of physical neurons, a single hardware pipeline processes neuron state stored in external memory. The pipeline reads one neuron's state from SDRAM, computes synaptic input by summing weighted spikes from connected neurons, applies the Izhikevich differential equations to update membrane voltage, checks for threshold crossing, and writes the updated state back to memory. This cycle repeats for every neuron in the network.

Figure 3: Block diagram comparing fully parallel vs time-multiplexed architecture

Intel's Loihi and Loihi 2 neuromorphic research chips use this exact approach. Each Loihi 2 neuromorphic core does not contain 8,192 physical neurons. Instead, a single update engine iterates through neuron states stored in on-chip SRAM, processing each neuron sequentially at high speed. This is the established industrial approach to neuromorphic computing, not a compromise specific to this implementation.

For this design, I target a 50 megahertz clock frequency based on Quartus Prime timing estimates for the individual neuron and synapse pipelines. At that rate, the architecture is designed to perform on the order of one million neuron state updates per second. For a network of 1,000 neurons, each neuron would receive one update every millisecond, which sits comfortably inside biologically realistic timescales where typical cortical firing rates range from 1 to 100 hertz. Full-chip compilation and timing closure are still in progress, so these numbers describe the intended operating regime rather than final silicon measurements.

Resource allocation across the design breaks down as follows. The time-multiplexed neuronal sheet, which is the core computational fabric, consumes approximately 10,200 logic elements based on the final Quartus compilation report. A global modulator neuron providing homeostatic control uses roughly 980 logic elements. Infrastructure including the VGA controller, SDRAM interface controller, and top-level control logic accounts for 3,400 logic elements. Approximately 800 logic elements remain unused, providing headroom for future enhancements or timing optimization.

Figure 4: Resource utilization from Quartus Prime

The implementation uses the Izhikevich neuron model, which balances biological realism with computational efficiency. Unlike simple integrate-and-fire models that treat neurons as threshold switches, or complex Hodgkin-Huxley models requiring multiple coupled differential equations, the Izhikevich model captures essential spiking dynamics through two coupled equations governing membrane voltage and recovery.

The model equations are solved using fixed-point arithmetic with 16-bit precision for state variables and 12-bit precision for parameters. Euler integration with a timestep of 1 millisecond provides acceptable accuracy for the target application while remaining implementable in hardware. More sophisticated integration methods like Runge-Kutta would improve accuracy but require multiple evaluation steps per update, which would reduce the effective network size given the time-multiplexing constraint.

Figure 5: Oscilloscope trace showing actual spike waveforms

Parameter values for the Izhikevich model determine spiking behavior. For this implementation, all neurons use regular spiking parameters similar to cortical excitatory neurons, with a parameter value of 0.02, b parameter of 0.2, c parameter of negative 65, and d parameter of 8. These values produce realistic spike timing and adaptation behavior without requiring per-neuron parameter customization in the current version.

Testing the isolated neuron pipeline with constant input current confirms proper dynamics. Applying input current of 10 units produces regular spiking at approximately 8 hertz. Increasing current to 20 units increases firing rate to roughly 15 hertz. Removing input current causes firing to cease within 50 milliseconds as the membrane voltage decays below threshold. This matches expected Izhikevich model behavior.

Network Architecture and Synaptic Connectivity

The network topology uses a two-dimensional grid arrangement matching the spatial structure of the input images. For the current implementation, a 16 by 16 grid provides 256 neurons corresponding to 256 pixels. Each neuron connects to its spatial neighbors within a radius of 3 pixels, creating local connectivity similar to lateral connections in visual cortex.

Synaptic weights determine the strength of connection between neurons. In biological systems, these weights change through learning and experience. For this implementation, weights are calculated offline using a Hebbian learning principle. Neurons corresponding to pixels that frequently appear together in training images develop strong mutual connections. This creates an associative memory where the complete pattern becomes a stable attractor state in the network dynamics.

Figure 6: Visualization of synaptic weight matrix

The weight calculation process begins with training fragments. I manually created 20 cropped portions of the target image, each showing different regions with 40 to 60 percent of the complete pattern. For each fragment, a simple Hebbian rule strengthens connections between active neurons. If neurons i and j are both active in a fragment, the weight connecting them increases by a small amount. After processing all fragments, the weight matrix is normalized and thresholded to eliminate weak connections.

This offline training produces a 256 by 256 synaptic weight matrix requiring 65,536 weight values. With 8-bit precision per weight, the matrix consumes 64 kilobytes of storage. The SDRAM provides ample space for this data along with neuron state variables.

During operation, the synaptic computation pipeline reads the current spike state of all neurons, multiplies each spike by the corresponding synaptic weight, and sums these contributions to calculate total input current for the target neuron. This sum becomes the input to the Izhikevich differential equations for that timestep.

The hardware implementation relies on a specific state machine within the `synapse_core.v` module to manage the multiply-accumulate process. Because the Cyclone IV has limited multipliers (only 56 embedded 9-bit elements), we cannot compute all inputs in parallel. Instead, for each target neuron update, the pipeline enters a `FETCH_SPIKE_VECTOR` state, reading the 256-bit spike history register from the previous 100ms timeframe. It then transitions to `ACCUMULATE`, iterating through a counter from 0 to 255.

A simple boolean AND gate checks the specific bit in the spike vector corresponding to the current counter index. If the bit is `0`, the pipeline asserts a `nop` (no operation) signal. If the bit is `1`, it triggers a read request to the weight matrix stored in SDRAM at `base_addr + (target_neuron_id * 256) + source_neuron_id`. This event-driven gating is physical, not simulated; the `mac_enable` signal on the DSP block stays low unless a spike is present, preventing dynamic power switching in the adder logic.

Figure 7: Logic analyzer trace of the synaptic pipeline. Channel 0 (spike_valid) indicates a valid presynaptic spike bit. Channel 1 (weight_valid) goes high 6 clock cycles later, reflecting SDRAM CAS latency. Channel 2 (mac_valid) pulses only when both inputs align, confirming that arithmetic operations are physically gated by spike presence.

Homeostatic Stability of Electrical Impulses and Overall Regulation

Spiking neural networks can exhibit instability without proper regulation. Positive feedback through strong recurrent connections can cause runaway excitation where activity escalates uncontrollably. Biological networks prevent this through inhibitory interneurons and neuromodulatory systems that adjust excitability based on overall activity levels.

This implementation includes a global modulator neuron operating in parallel with the main neuronal sheet. The modulator monitors average firing rate across the network by counting total spikes per timestep. When activity exceeds a threshold of 80 spikes per millisecond, indicating excessive excitation, the modulator reduces bias current to all neurons. When activity falls below 20 spikes per millisecond, the modulator increases bias current to maintain ongoing dynamics.

This homeostatic control mechanism consumes approximately 980 logic elements as mentioned earlier and operates on a 10 millisecond timescale. The modulator updates its bias adjustment every 10 milliseconds based on accumulated spike counts, preventing rapid oscillations while maintaining responsiveness to sustained activity changes.

Testing shows this mechanism successfully prevents runaway excitation during network operation. Initial tests without the modulator resulted in either quiescent networks with no activity or explosive activity that saturated all neurons within 100 milliseconds. Adding the modulator allows the network to maintain stable dynamics with average activity levels between 40 and 60 spikes per millisecond.

The evaluation protocol is defined as follows, even though it has not yet run on the FPGA due to the compilation issues described earlier. The corrupted pattern will be loaded into the network by setting initial membrane voltages high for active pixels and low for inactive pixels. Success will be defined as achieving greater than 85 percent pixel agreement between the network output and the original uncorrupted pattern. The hardware implementation is intended to reproduce this regime on the AC415 board.

I also intend to sweep the corruption level. At lower corruption levels (for example, 20–30 percent pixels removed), I expect higher success rates and shorter convergence times. As corruption increases beyond 50–60 percent, it becomes more likely that the initial state lies outside the basin of attraction of the learned pattern, and the network may converge to an incorrect attractor or fall into oscillatory dynamics. These “failure cases” are scientifically interesting, because they reveal where the learned energy landscape is too shallow or where additional structure (such as inhibitory interneurons) is required.

The Efficiency and Event-Driven Architecture of Non-PU

One of the central motivations for the NonPU architecture is event-driven efficiency: the idea that computation and power draw should scale with real informational content in the input, not with a fixed matrix size. The current hardware bring-up state does not yet allow me to publish final watt-meter readings from the AC415 board, but the measurement methodology and comparison framework are already defined.

In the planned setup, power measures will be taken using a digital multimeter or inline power monitor connected directly to the AC415 FPGA supply rails. Three operating conditions define the test protocol beginning with Configured Idle where the bitstream is loaded and clocks are running but no external input activity or spike stimulation occurs. Moderate Activity involves input patterns with sparse spikes or low-contrast structure while High Activity introduces heavily corrupted patterns or dense random input that drives widespread spiking. For each case, the NonPU serves as a comparison against a conventional fixed-topology feedforward artificial neural network implemented on the same FPGA or a small GPU. The key quantity of interest is not raw speed but whether power consumption in the spiking system visibly tracks input activity. A mostly black image should produce close-to-idle power draw whereas a dense or noisy image should draw more current as more neurons spike and more synapses are exercised.

Exact watt values and scaling factors are intentionally omitted here, because they would be speculative until the bitstream compiles and the design runs stably on real hardware. Published GPU board powers (for example, 75 watts for an entry-level card) provide an approximate reference for the orders of magnitude involved, but the true value of this project is demonstrating the shape of the power curve: flat for matrix engines, activity-dependent for spiking hardware.

Engineering Implementation

The complete Verilog HDL implementation is available on GitHub. The following modules comprise the NonPU architecture, each implementing specific components of the neuromorphic processing system. All source code is available at github.com/nekshadesilva/nonpu on the main branch.

activitymonitor.v

The neuron sheet produces spikes. This module converts that stream into structured information about regional activity. Each tile in the 16×16 grid has a counter. When spike_valid fires, the module translates neuron index to tile position and increments that tile's counter. At frame_tick, the monitor scans all tiles, compares counts to threshold, and emits events for overactive or underactive regions. Counters reset. This is where spatial awareness exists.

fb_ram.v

Dual-port RAM. One side connects to spike2letter, which writes pixel values as neurons update. The other connects to display controller or testbench. Each memory entry holds one 8-bit pixel. Address equals neuron index. Neuron 0 is pixel 0. For a 16×16 sheet, depth is 256. The sheet writes new activity while another process reads the current image simultaneously. No arbitration required. True dual-port.

neuronalpipes.v

This module is one neuron in silicon. v_in and u_in represent membrane voltage and recovery variable from the previous timestep. I_in is total current from image stimulus and synaptic inputs. The core implements Izhikevich equations using fixed-point arithmetic. When voltage crosses threshold, spike_out fires for one clock cycle, voltage resets, and recovery adjusts. This block does not classify. Unlike dense matrix multiplication where every weight is computed every cycle, synaptic operations here are sparse and event-gated—only active connections contribute. It solves discrete differential equations in hardware.

sheet.v

The FPGA does not contain 256 separate neuron blocks. This module uses one neuronalpipes core and time-divides it across a 16×16 grid. Memory arrays store voltage, recovery, and input current for each neuron. A counter selects one neuron per cycle, reads its state, feeds it through the core, and writes updated values back. When a neuron fires, spike_valid asserts with the neuron index. When the counter wraps from neuron 255 to 0, frame_tick pulses. One physical core multiplexed over 256 logical neurons.

spike2letter.v

Humans cannot read spike trains. This module translates neuron activity to visible pixels. For each neuron update, it receives spike_valid, neuron_idx, and voltage v_in. If the neuron spiked, write 255 to framebuffer at that neuron's address. If not, rescale voltage to 8-bit grayscale and write that. Output signals fb_we, fb_addr, and fb_data connect directly to framebuffer RAM. In hardware, these values become monitor pixels. In simulation, they form an image of emergent network patterns.

superneuron.v

This is not another neuron. This is a control plane above the sheet. It maintains one bias value per tile in memory. When activitymonitor reports a tile firing excessively, this module reduces that tile's bias. Underactive tiles get increased bias. The internal variable G represents global gain, adapting based on overall activity. tile_id_query from the sheet indicates which tile the current neuron belongs to. The module looks up that tile's bias and outputs it as bias_out. The sheet adds this to neuron input current. Spikes shape biases. Biases shape spikes. Closed loop homeostasis.

All modules are synthesized using Intel Quartus Prime Lite Edition and target the Cyclone IV EP4CE15 FPGA. The complete source code repository includes additional documentation, testbenches, and synthesis scripts. For detailed implementation information, refer to the GitHub repository.

Comparison with Past Neuromorphic Innovations

Intel's Loihi and Loihi 2 represent the current state of industrial neuromorphic computing research. Loihi 2, announced in 2021, contains 128 neuromorphic cores each supporting up to 8,192 neurons through time-multiplexing as described earlier. The chip integrates approximately 2.3 billion transistors and implements sophisticated learning rules including spike-timing-dependent plasticity directly in hardware.

This project does not attempt to compete with Loihi at the scale or sophistication level. Loihi 2 is a custom ASIC designed by a team of specialized engineers with access to advanced fabrication processes. This implementation runs on a commodity development board costing approximately 50 dollars.

The value lies in demonstrating that core neuromorphic principles scale down to minimal hardware. Time-multiplexed spiking neurons, event-driven computation, and attractor dynamics can be implemented from first principles on constrained FPGAs. This validates the fundamental architecture and provides a platform for education and experimentation.

IBM's TrueNorth chip, released in 2014, takes a different architectural approach using fully parallel neurons rather than time-multiplexing. TrueNorth contains 4,096 neuromorphic cores with 256 neurons per core, giving 1 million neurons total implemented in parallel hardware. This enables extremely low latency but requires a massive silicon area of 4.3 square centimeters in 28 nanometer technology.

BrainScaleS, developed at Heidelberg University, uses analog circuits to simulate neuron dynamics up to 10,000 times faster than biological real-time. This accelerated simulation enables rapid exploration of network configurations but requires careful calibration and suffers from parameter variability inherent in analog circuits.

Limitations and Known Issues Currently Being Solved

Several limitations and simplifications exist in the current implementation that warrant explicit acknowledgment. The network uses only excitatory connections. Biological cortex contains approximately 20 percent inhibitory interneurons that shape network dynamics through precisely timed suppression. Adding inhibitory neurons would improve stability and enable more complex computational patterns but would require additional logic resources and more sophisticated weight matrix structure.

Synaptic plasticity occurs offline. The weights are pre-calculated and loaded into memory before network operation. True neuromorphic systems implement spike-timing-dependent plasticity in hardware, allowing weights to adapt continuously based on observed spike patterns. The current architecture includes placeholder logic for STDP computation but this remains unimplemented pending future development.

The pattern completion task uses only a single learned pattern. Testing with multiple stored patterns would reveal capacity limits and interference effects inherent in attractor network dynamics. Preliminary testing with two patterns shows successful retrieval but with reduced robustness compared to single-pattern operation.

Timing closure at higher clock frequencies is expected to be challenging. The initial design target is 100 megahertz operation, but early synthesis reports and back-of-the-envelope path estimates suggest that critical paths in the multiplier chain will likely exceed 10 nanoseconds without additional pipelining. For this reason, the current NonPU design is scoped around a more conservative 50 megahertz clock, which should be attainable once the full top-level build compiles cleanly. Further pipeline optimization and restructuring of the arithmetic datapaths may allow higher operating frequencies in future revisions.

The VGA output provides only binary visualization. Adding grayscale rendering based on membrane voltage or firing rate would give more informative feedback about network state but would require additional video memory and more complex rendering logic.

Testing coverage remains limited to controlled laboratory conditions. The system has not been validated under temperature variation, power supply noise, or long-duration operation exceeding several hours. Extended reliability testing would be necessary before deploying such a system in practical applications.

The NonPU Just Got Started.

Several concrete enhancements are planned for future versions, each with specific implementation pathways. Expanding spatial resolution to 32 by 32 pixels would quadruple the network size to 1,024 neurons. The current time-multiplexing architecture supports this directly by extending the neuron iteration counter and allocating additional SDRAM for state storage. The synaptic weight matrix would grow to 1 megabyte, well within available memory. Update rate would decrease to approximately 1.5 milliseconds per neuron cycle, which remains within acceptable biological timescales.

Implementing on-chip spike-timing-dependent plasticity requires adding logic to detect spike coincidences and update synaptic weights. The weight update can occur asynchronously during the inter-neuron pipeline gaps when the main processing pipeline is idle. STDP computation adds approximately 1,500 logic elements based on preliminary synthesis experiments, which fits within the available headroom.

Adding inhibitory interneurons with 20 percent probability in the grid would require distinguishing excitatory and inhibitory neuron types in the state storage and implementing sign inversion for inhibitory synaptic weights. This is architecturally straightforward and adds minimal logic overhead but significantly enriches network dynamics.

Migrating to a larger FPGA platform like the Cyclone V or Cyclone 10 would provide 10x to 50x more logic elements, enabling either fully parallel implementation of hundreds of neurons or time-multiplexed simulation of networks with 10,000 to 100,000 neurons. This would enable more complex computational tasks beyond simple pattern completion.

Interfacing with actual biological neural tissue represents the long-term research goal. Cortical organoids or brain slice preparations generate electrical signals detectable through microelectrode arrays. Bidirectional communication between the FPGA spiking network and biological tissue would enable hybrid bio-electronic systems. This requires specialized analog front-end hardware not present on the current development board but represents a feasible extension with appropriate instrumentation.

Figure 16: NonPU v0.1 runs on the AC415 Cyclone IV board as a 16×16 time-multiplexed spiking sheet that performs pattern completion in pure silicon. The next stage, version 0.2, migrates this architecture to an Intel Altera Cyclone V, scaling to at least a 32×32 sheet. Please note that this innivation can scale indefinitely for many versions.
